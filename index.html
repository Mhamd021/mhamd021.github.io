Waitless — Digital Queue System

  • NestJS
  • Bull Queue
  • Socket.IO
  • PostgreSQL
  • Prisma v7
  • Next.js
  • Resend
  • Docker
Real-time Queue Management Customers join via a link: no app, no login required. Position updates instantly.
Bull Queue + Resend Email notifications are queued as Bull jobs for reliable delivery, even if the server restarts.
Socket.IO Real-time All connected clients update instantly when admin calls next, with no polling needed.
JWT + Secret Key Auth Admin is protected by JWT. Registration requires a secret key, so there is no open sign-up.
Customer Token No login for customers. A UUID token acts as their personal tracking link.
Docker Compose PostgreSQL and Redis run in containers, so one command starts the full stack.

Built for small businesses like barbers, clinics, and restaurants to replace physical waiting lines with a digital real-time system.

  • Real-time position updates via Socket.IO with no refresh needed.
  • Bull Queue for reliable email notifications that survive server restarts.
  • No customer login. UUID token works as the tracking link.
  • Admin dashboard with live queue view, call next, and open/close controls.
  • Prisma v7 with pg adapter for a modern ORM setup.
  • Multi-queue support: one admin, multiple queues.
